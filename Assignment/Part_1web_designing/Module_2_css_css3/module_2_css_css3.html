(Q.5) What do you understand by CSS opacity?

Ans :

The opacity CSS property sets the opacity of an element.
Opacity is the degree to which content behind an element is hidden, and is the opposite of transparency.

Example:

(Q.8) What is the use of the background-position property?

Ans :

The background-position property sets the starting position of a background image. Tip: By default, a background-image is placed at the top-left corner of an element, and repeated both vertically and horizontally.

Example:

/* Keyword values */ background-position: top;
background-position: bottom;
background-position: left;
background-position: right;
background-position: center;

background-position: 20% 70%;

/* absolute length values */
background-position: 0 0;

/* Multiple images */
background-position: 0 0, center;

/* Edge offsets values */
background-position: bottom 10px right 20px;
background-position: right 3em bottom 10px;
background-position: bottom 10px right;
background-position: top right 10px;

(Q.10) Why should background and color be used as separate properties?

Ans :

Using background and color as separate properties in CSS allows for greater control and flexibility when styling elements. Each property targets a specific aspect of an element's appearance, making it easier to apply and manage styles.

1.   Separation of concerns:
              background and color are separate properties because they target different aspects of an element's appearance.
              background is responsible for setting the background color, image, position, size, and other properties of an element.
              color sets the color of the text content within an element. By keeping these properties separate, you can independently manage
              the background and text color of an element without affecting the other.

2.  Readability and maintainability:
              Separating these properties makes it easier to read and maintain your code.
              When you have clear, distinct properties for each aspect of an element's style, it's easier to understand what each line of CSS
              is doing. This makes it simpler to troubleshoot and modify existing code.

3.  Flexibility:
              Using separate properties for background and color allows you to apply specific styles to different
              parts of your elements. For example, you may want to change the background color of a button when it's hovered over,
              but keep the text color the same. By keeping these properties separate, it's easy to modify one without affecting the other.

Example:

/* Using separate properties */

.button {
       background: #ff9900;
       color: #ffffff;
}

.button:hover {
       background: #cc7a00;
}



/* Using a shorthand property */

.button {
       background-color: #ff9900;
       color: #ffffff;
}

.button:hover {
       background-color: #cc7a00;
}

(Q.11) How to center block elements using CSS1?

Ans :

CSS1 does not provide a direct way to center block elements horizontally or vertically. However, you can achieve horizontal centering by setting the left and right margins of the block element to "auto". For example

Example:

<div style="width: 50%; margin-left: auto; margin-right: auto; background-color: #ccc; color: #111; padding: 7px 50px;">
<p>This block element is horizontally centered.</p>
</div>

<div class="p-5" style="height: 200px; padding-top: 50px; padding-bottom: 50px; background-color: #ccc;">
<div style="height: 100px; margin-top: auto; margin-bottom: auto; background-color: #fff;">
<p>This block element is horizontally centered.</p>
</div>

(Q.12) How to maintain the CSS specifications?

Ans :

1. Use a consistent naming convention:
              Develop a naming convention for your CSS classes and stick to it.
              This will make it easier to understand and maintain the codebase over time.

2. Comment your code:
              Add comments to your CSS code to explain what each section does.
              This will make it easier to understand the code and make changes later on.

3. Use version control:
              Use a version control system such as Git to track changes to your CSS files.
              This will allow you to easily revert to previous versions if needed.

4. Minimize the use of !important:
              Try to avoid using the !important declaration in your CSS code as it can make it difficult
              to override styles later on.

5. Use a preprocessor:
              Consider using a CSS preprocessor like Sass or Less to make it easier to maintain your code.
              Preprocessors allow you to write cleaner and more organized code, and can save you time in the long run.

6. Document your style guide:
              Develop a style guide for your project and document it in a central location.
              This will help ensure consistency across your codebase and make it easier to maintain the project over time.

7. Regularly audit your code:
              Regularly review your CSS code to identify any outdated or unnecessary styles.
              This will help keep your codebase lean and easy to maintain.

(Q.13) What are the ways to integrate CSS as a web page?

Ans :

CSS may be added to HTML in three different ways.

1.  (External CSS) :
              Each HTML page must include a reference to the external style sheet file inside the <link> element, inside the head section.

2.  (Internal CSS) :
              An internal style sheet may be used if one single HTML page has a unique style.
              The internal style is defined inside the <style></style> element, inside the head section.

3.  (Inline CSS) :
              An inline style may be used to apply a unique style for a single element.

Example:

1.   (External CSS) :
<head>
       <link rel="stylesheet" type="text/css" href="stylesheet.css">
</head>

2.   (Internal CSS) :
<head>
<style>
body {
       <background-color: linen;
}

h1 {
       <color: maroon;
       <margin-left: 40px;
}
</style>
</head>

3.   (Inline CSS) :
<!DOCTYPE html>
<html>
<body>
       <h1 style="color:blue;text-align:center;">This is a heading</h1>
       <p style="color:red;">This is a paragraph.</p>
</body>
</html>

(Q.17) What is the meaning of the CSS selector?

Ans :

A CSS selector is the first part of a CSS Rule. It is a pattern of elements and other terms that tell the browser which HTML elements should be selected to have the CSS property values inside the rule applied to them.

       color: red;
}

(Q.18) What are the media types allowed by CSS?

Ans : CSS2 Introduced Media Types :

The @media rule, introduced in CSS2, made it possible to define different style rules for different media types.

You could have one set of style rules for computer screens, one for printers, one for handheld devices, one for television-type devices, and so on. Unfortunately these media types never got a lot of support by devices, other than the print media type.

Example:

/* Max-width */
@media only screen and (max-width: 600px) {...}

/* Min-width */
@media only screen and (min-width: 600px) {...}

/* Combining media query expressions */
@media only screen and (max-width: 600px) and (min-width: 400px) {...}
